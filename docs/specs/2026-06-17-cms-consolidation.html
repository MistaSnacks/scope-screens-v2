The rule of thumb

Fold in

Becomes fields on the parent section record
  • Small — roughly 3–5 entries
  • Fixed — count rarely changes
  • Belongs to one section — not global
  • Text only — no per-row image/file upload

Stay a collection

Keeps the spreadsheet / add-rows workflow
  • Variable length — editors add/remove often
  • Media or file per row — image / URL / download
  • Reused across more than one place

Execution (per folded collection)

Mirrors the Built For Access trial, for each of Socials, Clapboard, SubmissionChips, GivingTiers:

  1. Add the new fields to the parent record (create-field).
  2. Copy current values from the source collection into the parent record.
  3. Reorder so the folded fields sit at the bottom of the form.
  4. Delete the now-empty source collection.
CMS-only for now (your call #2). These collections are read by main's lib/site-content.ts + components; you're on ALT. After the fold, main's sections will fall back to their hard-coded defaults until we wire them to the new fields (e.g. read support.givingTier1Label instead of the GivingTiers list). We'll do that wiring together when you're ready. Values are copied into the parent record before any delete, so nothing is lost.
